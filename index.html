Viaduct

Dev & maintainer

A GraphQL-based system that gives services a single, unified interface to any data source. Rather than every team wiring up its own access patterns, Viaduct puts one highly connected schema in front of the whole estate — a data-oriented service mesh. Open-sourced by Airbnb and written in Kotlin.

  • Kotlin
  • GraphQL
  • Service mesh
  • Schema design
github.com/airbnb/viaduct

GraphDSL

Creator

Generates type-safe Kotlin DSL query builders straight from your .graphqls schema files. No hand-written query strings, no extra serialisation layer — drop in the Gradle plugin, put your schema in src/main/graphql/, and the builders appear at compile time.

  • Kotlin
  • Code generation
  • Gradle plugin
  • Type safety
github.com/graphdsl/graphql-kotlin-dsl

Approach

How I like to build

01

Make the compiler do the arguing

If a mistake can be caught at build time, it should never make it to a test, let alone production. That instinct is most of why GraphDSL exists.

02

Boundaries before frameworks

Clean and hexagonal architecture aren't ceremony — they're what lets a codebase survive its third rewrite of the transport layer. Domain first, plumbing second.

03

One schema beats fifty integrations

Working on Viaduct sharpened this: the cost of a distributed system is rarely the services, it's the N×N ways they talk to each other.

04

Build it in the open

Public code gets better review, better docs and a longer life. It's also the fastest way I've found to learn from people better than me.
